Angular, standalone components allow you to build Angular applications without the need for NgModules. Introduced in Angular 14 and improved in subsequent versions, standalone components simplify application structure and enable modular development.

What Are Standalone Components?

Self-contained units: Standalone components can declare their dependencies directly and don’t require being part of an NgModule.
A signal represents a reactive value that automatically updates any dependent computations when its value changes.
An interceptor in Angular is a special type of service that allows you to intercept and modify HTTP requests or responses globally.
RxJS is a library for reactive programming using Observables. It allows you to work with asynchronous data streams, making it ideal for handling events, HTTP requests, and complex data transformations in Angular applications.
NgRx is a state management library for Angular based on Redux principles. It uses RxJS under the hood for managing side effects and streams.
Hoisting in JavaScript is a behavior where variable and function declarations are moved to the top of their containing scope during the compilation phase.
A Resolver in Angular is a service that fetches or prepares data before navigating to a route. It ensures the required data is available for the component to use when it loads.
A Promise represents a single future value that resolves or rejects once. It’s primarily used for handling one-time asynchronous tasks like fetching data from an API.
An Observable is a lazy, multi-value stream that allows you to work with data over time. Observables are part of the RxJS library and are used extensively in Angular.
Dependency Injection (DI) is a design pattern and a core feature of Angular. It allows components, services, or other classes to request dependencies from an external source.
Directives are classes that add behavior to elements in your Angular applications. They are one of the core building blocks of Angular.
Event Bubbling is a concept in event handling in which an event starts from the target element and propagates upward through its ancestors in the DOM hierarchy.
